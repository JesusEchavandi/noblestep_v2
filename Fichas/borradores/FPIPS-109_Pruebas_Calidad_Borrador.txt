2. INTRODUCCION
Documento de validacion funcional y tecnica para asegurar que NobleStep cumple requerimientos de negocio y calidad.

3. OBJETIVOS

8. CASOS DE PRUEBA DETALLADOS
CP01 - Login valido
- Requisito: RF01
- Precondicion: usuario activo existente.
- Pasos: ingresar credenciales validas y enviar.
- Resultado esperado: acceso exitoso y token valido.

CP02 - Login invalido
- Requisito: RF01
- Precondicion: usuario existente.
- Pasos: ingresar password incorrecta.
- Resultado esperado: mensaje de credenciales invalidas.

CP03 - Solicitar recuperacion
- Requisito: RF02
- Pasos: registrar correo valido en olvido de contrasena.
- Resultado esperado: respuesta OK y correo enviado.

CP04 - Reset con token valido
- Requisito: RF02
- Precondicion: token vigente.
- Pasos: abrir enlace y registrar nueva contrasena.
- Resultado esperado: contrasena actualizada.

CP05 - Reset con token expirado
- Requisito: RF02
- Precondicion: token vencido.
- Pasos: intentar cambio de contrasena.
- Resultado esperado: mensaje token invalido o expirado.

CP06 - Catalogo visible
- Requisito: RF03
- Pasos: abrir tienda y navegar catalogo.
- Resultado esperado: listado correcto con variantes.

CP07 - Filtro por categoria
- Requisito: RF03
- Pasos: aplicar filtro por categoria.
- Resultado esperado: solo productos de categoria seleccionada.

CP08 - Confirmar pedido con stock
- Requisito: RF04
- Precondicion: stock suficiente.
- Pasos: agregar al carrito y confirmar.
- Resultado esperado: pedido creado y stock reducido.

CP09 - Confirmar pedido sin stock
- Requisito: RF04
- Precondicion: stock insuficiente.
- Pasos: intentar confirmar.
- Resultado esperado: rechazo y mensaje de stock insuficiente.

CP10 - Integridad pedido-detalle
- Requisito: RF04
- Pasos: confirmar pedido multi-item.
- Resultado esperado: cabecera y detalle consistentes.

CP11 - Crear producto
- Requisito: RF05
- Actor: admin
- Resultado esperado: producto registrado correctamente.

CP12 - Editar variante
- Requisito: RF05
- Actor: admin
- Resultado esperado: cambios persistidos en variante.

CP13 - Ingreso por compra
- Requisito: RF06
- Resultado esperado: stock incrementado segun detalle compra.

CP14 - Salida por venta
- Requisito: RF06
- Resultado esperado: stock decrementado sin quedar negativo.

CP15 - Cambiar estado pedido
- Requisito: RF07
- Actor: admin/operador
- Resultado esperado: estado actualizado con trazabilidad.

CP16 - Consulta historico pedidos
- Requisito: RF07
- Resultado esperado: listado ordenado y filtrable.

CP17 - Crear usuario (admin)
- Requisito: RF08
- Resultado esperado: usuario creado con rol asignado.

CP18 - Restriccion por rol
- Requisito: RF08
- Resultado esperado: usuario no admin no accede a gestion de usuarios.

CP19 - Reporte ventas por periodo
- Requisito: RF09
- Resultado esperado: totales consistentes con ventas registradas.

CP20 - Reporte sin datos
- Requisito: RF09
- Resultado esperado: reporte vacio sin error de aplicacion.
